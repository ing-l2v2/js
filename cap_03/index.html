3. Programación Orientada a Objetos

3.1. Prototipos

JS es un lenguaje multiparadigma: programación funcional y orientada a objetos. En POO hay clases (modelo a seguir para generar instancias), objetos (atributos y métodos) son instancias de clases. JS es un lenguaje orientado a objetos basado en prototipos no en clases. El compilador convierte las clases en funciones prototípicas.
Función constructora - se la construye una vez y a partir de esta se generan nuevas instancias u objetos. Los atributos como los métodos deben colgar de this. 

    function AnimalPrototipo(nom, genero) {
      this.nombre = nom;
      this.genero = genero;
      this.sonar = function () {
        console.log(`Hago sonidos porque estoy vivo, soy ${this.genero}.`);
      };
      this.saludar = function () {
        console.log(`Hola me llamo ${this.nombre}`);
      };
    }
    const tom = new AnimalPrototipo("Tom", "macho"),
      mia = new AnimalPrototipo("Mia", "Hembra");
    console.log(tom, mia);
    tom.sonar();              tom.saludar();
    mia.sonar();              mia.saludar();

Si se requieren muchos objetos se generarían muchas copias de los métodos y provocaría mayor uso de memoria, una mejora sería asignar al prototipo de la función constructora los métodos y los métodos quedar pegados al prototipo de la función constructora, provocando mejora significativas en rendimiento y memoria. 

    function AnimalPrototipo(nom, genero) {
      this.nombre = nom;
      this.genero = genero;
    }
    // Métodos agregados al Prototipo de la función constructora de AnimalPrototipo
    AnimalPrototipo.prototype.sonar = function () {
      console.log( `Hago sonidos porque estoy vivo, soy ${this.genero}.` );
    };
    AnimalPrototipo.prototype.saludar = function () {
      console.log(`Hola me llamo ${this.nombre}`);
    };
    const tom = new AnimalPrototipo("Tom", "macho"),
      mia = new AnimalPrototipo("Mia", "Hembra");
    console.log(tom, mia);
    tom.sonar();    tom.saludar();
    mia.sonar();    mia.saludar();

3.2. Herencia Prototípica

La función prototípica que heredará de otra función prototípica recibe los mismos parámetros más los propios 

    function Perro(nombre, genero, tamanio, enfermo) {
      this.fncSuper = AnimalPrototipo;
      this.fncSuper(nombre, genero);
      this.tamanio = tamanio;      this.estaEnfermo = enfermo;
    }
    
    // Perro esta heredando de AnimalPrototipo
    Perro.prototype = new AnimalPrototipo();
    Perro.prototype.constructor = Perro;
    
    // Sobreescritura de métodos
    Perro.prototype.sonar = function () {
      console.log( `Soy un perro ${this.genero} y hago guuuau guuuuau.` );
    };

    // Método própio
    Perro.prototype.fcEnfermo = function () {
      if (this.estaEnfermo) { console.log("Me siento enfermito");
      } else { console.log("Estoy muy saludable !!!"); }
    };
    
    const tomy = new Perro("Tom", "macho", "pequeño", false);
    console.log(tomy);
    tomy.sonar();    tomy.saludar();    tomy.fcEnfermo();

3.3. Clases y Herencia

A partir del 2015 el estandar de JS provee las clases. Pero toda clase al estar JS basado en prototipos entonces lo compila a prototipos.

Las clase no reciben parámetros, tienen un método especial llamado constructor el cual recibe las propiedades. Los métodos no son necesarios sacarlos para ubicarlos en el prototipo dado que JS ya hace esa mejora. 

    class AnimalClass {
      constructor(nombre, genero) {
        this.nombre = nombre;        this.genero = genero;
      }
      sonar() {
        console.log(`Hago sonidos porque estoy vivo, soy ${this.genero}.`);
      }
      saludar() { console.log(`Hola me llamo ${nombre}`); }
    }
    const tomClass = new AnimalClass("Tom", "macho");
    tomClass.sonar();
    console.log(tomClass);

La herencia de clase, observar la palabra extends, dentro del constructor participa el método super, existe la sobreescritura de métodos en el ejemplo el método sonar 

    class PerroClass extends AnimalClass {
      constructor(nombre, genero, tamanio, enfermo) {
        // super es un método que manda a llamar al constructor de la clase padre
        super(nombre, genero);
        this.tamanio = tamanio;
        this.enfermo = enfermo;
        this.raza = null;
      }
      // Sobreescritura de métodos
      sonar() { console.log("Soy un perro y realizo ladridos"); }

      // Métodos propios
      ladrar() { console.log("Guauuuu Guauu!!!"); }
      
      // Método estático
      static queEres() {
        console.log( "Los perros somo animales mamiferos que pertenecemos a la familia de los caninos. Somos considerados los mejores amigos del hombre." );
      }
      // Métodos getter y setter
      get getRaza(){ return this.raza; }
      set setRaza(raza){ this.raza = raza; }      
    }
    const pitibull = new PerroClass(
      "MiniBull", "Hembra", "gigante", true
    );
    console.log(pitibull);
    pitibull.ladrar();

3.4. Métodos estáticos, getters y setters

No se puede crear una clase privada en JS, todas las clases son públicas

Un método estático, se puede ejecutar sin necesidad de instanciar la clase, un método se lo define como estático anteponiendo la palabra static. En el subtema previo se observa la declaración de un método estático llamado queEres() su invocación sería: 

    PerroClass.queEres();

Los setters y getters son métodos especiales que permiten establecer y obtener los valores de atributos de la clase.
El atributo this.raza = null; a este se le definen dos métodos 

    get getRaza(){ return this.raza; }
    set setRaza(raza){ this.raza = raza; }

getRaza y setRaza se lo invoca como a un atributo no como a método 

    pitibull.setRaza = "Grán Danés";
    console.log(pitibull.getRaza);
